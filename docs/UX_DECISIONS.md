# UX Decisions

Dream Fund LabsのDiscoveryで確定したUI/UX方針を記録する。

## 2026-07-17

### 採用：オンボーディング

初めて利用するユーザー向けに、オンボーディングを必須とする。

オンボーディングでは、少なくとも以下を説明する。

- Dream Fund Labsは「投資判断を育てる」アプリであること
- 月の投資予算を設定すること
- AIとのチャットで今月の投資方針を決めること
- 今月の投資方針は原則として1か月間維持すること
- ホームでは、前回からの変化と現在の投資判断を確認すること
- Dream20の役割と見方
- 基本的な操作方法

### 採用：投資方針を決めるチャット

ホーム内に相談文を展開するのではなく、専用のチャット画面へ遷移する。

ユーザーは自由記述で、今月気になっていることや希望を入力できる。

自由記述がない場合は、初期設定された情報、月の予算、保有銘柄などをもとにAIが方針案を提案する。

ユーザーが方針案を確認し、「この方針で確定する」を実行した時点で今月の投資方針として保存する。

### 採用：投資方針は簡単には変更できない

今月の投資方針は原則として1か月間維持する。

通常画面には編集導線を置かない。

変更機能は次のような深い階層に配置する。

設定
→ 投資設定
→ 今月の投資方針
→ 今月の投資方針を変更する

変更前には確認画面を表示し、変更が例外的な操作であることを伝える。

### 採用：ハンバーガーメニュー

画面上部にハンバーガーメニューを設置し、端からスライド表示するメニューを採用する。

想定メニュー：

- ホーム
- Dream20
- 使い方
- 設定

### 採用：使い方ページ

オンボーディングで説明した内容を、後からいつでも確認できる「使い方」ページを設置する。

主な内容：

- Dream Fund Labsとは
- 基本的な利用の流れ
- ホーム画面の見方
- Dream20とは
- 今月の投資方針とは
- 現在の投資判断とは
- 前回からの変化とは
- よくある質問

思想や哲学を押し付けるページにはせず、実用的な説明に限定する。

### 採用：画面ごとの「？」ヘルプ

各主要画面の右上に「？」を配置する。

押すと、現在表示している画面に関係する説明だけを表示する。

例：

- ホーム：投資方針、前回からの変化、現在の投資判断の説明
- Dream20：Dream20の選定目的と使い方
- 投資方針チャット：自由記述の使い方、方針確定後の扱い

### 見送り：AIチャットによるサイト内機能検索

現時点では、サイト機能をAIチャットで検索する機能は実装しない。

理由：

- ハンバーガーメニュー、使い方ページ、画面ごとの「？」で十分に案内できる
- 機能検索が必要になるほど画面構造を複雑にしない
- 投資相談チャットとサイト案内の役割を混ぜない

将来、機能数が大幅に増え、ユーザーが迷う状態になった場合に再検討する。
